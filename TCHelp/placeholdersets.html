Placeholders for sets of files

Placeholders for sets of files are used to deal with more than one file at a time. The following syntax is allowed for them:

$[q][s][f][r]<FILESET>

$
Starts the placeholder. Use "$$" to get "$".
q
All filenames will be separatly quoted using ".

s
All filenames will be issued in 8.3-format.

f
The forward slash '/' is used as directory separator.

r
The filenames will be given relative to the working directory of the project.
<FILESET>
One of the following:
TPF
All TeX-files of the project.
BPF
All BibTeX-files of the project.
GPF
All Graphic-files of the project.
TXC
All files generated by TeXnicCenter regarding the project. Currently only the .tcp and .tps files.
COF
All currently opened files in the editor.
APF
All project files. This is the sum of TPF, BPF, GPF and TXC.
AFS
All above filesets together. This is the sum of TPF, BPF, GPF, TXC and COF.

Examples:

$qTPF
Lists all (La)TeX-files reported by the structure parser of TeXnicCenter. Each quoted and separated from each other using a space, like

"D:\Temp\test\curvature3d.tex" "D:\Temp\test\definitions\macros.tex" "D:\Temp\test\chaptertwo.tex"


$fBPF
Lists all BibTeX-files reported by the structure parser of TeXnicCenter. The forward slash is used as directory separator. The files themselves are separated from each other using a space, like

D:/Temp/test/xbib.bib D:/Temp/test/morebibs/morexbib.bib


$COF
Lists all files, which are currently opened in the editor. They are separated from each other using a space (notice the problem with non-quoting of "Program Files"), like

D:\Temp\test\curvature3d.tex D:\Program Files\texmf\pdftex\latex\config\pdflatex.ini


$qsrAPF
Lists all files contained in the project (as reported by the structure parser of TeXnicCenter). Each quoted, filenames in 8.3-format and relative to the working directory and separated from each other using a space, like

"curvat~1.tcp" "curvat~1.tps" "curvat~1.tex" "xbib.bib" "morebibs\morexb~1.bib" "defini~1\macros.tex" "chapte~1.tex"


Application:

A good place to use the sets would be to define a tool like this:

Command: zip.exe
Arguments: a "%tm.zip" $qrAPF
Initial Directory: %dm

This will lead to a ZIP-archive, where all your files regarding the project are gathered in. You need to have a compression program (like WinZip or WinRAR) for this and to adjust the path and arguments to this program.

Note:

These placeholders will not generate duplicates of a filename. So you can always be sure, that each filename will be issued only once.

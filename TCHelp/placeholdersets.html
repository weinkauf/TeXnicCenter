Placeholders for sets of files

Allows the following syntax:

$[q][s][f][r]<FILESET>

$
Starts the Placeholder. Use "$$" to get "$".
q
Filenames will be quoted using ".

s
Filenames in 8.3-format.

f
Forward Slash '/' is used as Directory Separator.

r
Filenames will be given relative to the WorkingDir of the Project.
<FILESET>
One of the following:
TPF
All TeX-Files of the Project.
BPF
All BibTeX-Files of the Project.
GPF
All Graphic-Files of the Project.
TXC
All Files generated by TXC regarding the Project. Currently only the .tcp and .tps files.
COF
All currently opened Files in the Editor.
APF
All Project Files. This is the sum of TPF, BPF, GPF and TXC.
AFS
All above Filesets together. This is the sum of TPF, BPF, GPF, TXC and COF.

Examples:

$qTPF
Lists all TeX-Files reported by TXC-StructureParser. Each quoted and separated from each other using a space, like

"D:\Temp\test\curvature3d.tex" "D:\Temp\test\definitions\macros.tex" "D:\Temp\test\chaptertwo.tex"


$fBPF
Lists all BibTeX-Files reported by TXC-StructureParser. The Forward Slash is used as directory separator. The Files themselves are separated from each other using a space, like

D:/Temp/test/xbib.bib D:/Temp/test/morebibs/morexbib.bib


$COF
Lists all Files opened in the editor. Separated from each other using a space (notice the problem with non-quoting of "Program Files"), like

D:\Temp\test\curvature3d.tex D:\Program Files\texmf\pdftex\latex\config\pdflatex.ini


$qsrAPF
Lists all Files contained in the Project (as reported by TXC-StructureParser). Each quoted, Filenames in 8.3-Format and relative to the WorkingDir and separated from each other using a space, like

"curvat~1.tcp" "curvat~1.tps" "curvat~1.tex" "xbib.bib" "morebibs\morexb~1.bib" "defini~1\macros.tex" "chapte~1.tex"


Application:

A good place to use the sets would be:

zip.exe a "%tm.zip" $qsrAPF

The Set of FileNames are always unique - no duplicates of a filename.
